PhpIMinifier - Compress your Javascript and CSS files with PHP and YUI

Details
Created on Wednesday, 28 December 2011 21:01

This is a script that I've been meaning to write for a very long time. Ever since I read the book by Steve Souders - 'High Performance Web Sites' this has been on my mind, but didn't need anything like this since the websites I built in the past weren't quite this big.

Initially, I've created this to be used for a very specific Zend Framework application. However, I have made it generic enough and removed the ZF parts to avoid confusion.

So, what does this script do?
The PhpIMinifier is a PHP Directory Iterator that will compress all your css and js files using the YUI Compressor.

The PHP Iterator Minifier will take the following 5 steps:

  1. Iterate through all directories of chosen type
  2. Store each directory in a directory array
  3. Store each chosen type file in an array
  4. Create all chosen type directories in the new location
  5. Minify each chosen file type and copy it across to its new location

 

This is the directory structure I am using:

-----/public
--------------/js
-----------------/custom
---------------------------/folderA
----------------------------test1.js
----------------------------test2.js
---------------------------/folderB
----------------------------test3.js
---------------------------/folderC
----------------------------test4.js

After versioning, the above will look like this:

-----/public
--------------/js
-----------------/custom
---------------------------/newVersion
--------------------------------------------/folderA
---------------------------------------------test1.js
---------------------------------------------test2.js
--------------------------------------------/folderB
---------------------------------------------test3.js
--------------------------------------------/folderC
---------------------------------------------test4.js

The idea is that by having a different version on each deployment, we are able to set the caching for all files for more than a few weeks and if we happen to make any changes, as soon as we create a new version of a file then the browser will see this as a new file. So from a user's point, this will be much faster (and she won't have to clear cache every time she visits our site).

Let's take a look at the configuration.

Initially, we want to set the location of the YUI executable as well as the full path to the public directory.

$yuiLocation = "/Users/antonis/Sites/yui/build/yuicompressor-2.4.8pre.jar";
$public = "/Users/antonis/Sites/branches/development/public/";

Next we want to create a version. I use loosely the Ubuntu style versioning of YearMonth and also add v1 at the end, just in case I happen to deploy more than once on that particular month.

$version = "1112v1";

If you are using Zend Framework, you can automate the versioning by setting it in your application.ini and fetching it directly:

$configIni = parse_ini_file($public . "../application/configs/application.ini", TRUE); 
$version = $configIni["production"] ["version"];

We also need to set the root level of the JS and CSS directories:

$css = $public . "css/"; 
$js = $public . "js/custom/";

And that's the basic configuration done.
