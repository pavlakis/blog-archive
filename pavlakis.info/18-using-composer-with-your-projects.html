Using composer with your projects

Details
Created on Saturday, 30 March 2013 23:20

Composer is a great tool for managing the dependencies in your project.

I must admit, I’ve only just started using it and can already see how powerful and easy to use it is.

There are many good articles on how to set it up, so I’ll go over the basics on getting started in your own project with a basic Hello World example.

This is my directory structure: 

projectA
|__ App
|__ public

Let’s get composer using curl: 

curl -s https://getcomposer.org/installer | php 
sudo mv composer.phar /usr/local/bin/composer

From inside the root directory projectA, create composer.json with the following: 

{
    "require": {
    },
    "autoload": {
        "psr-0": {
            "App": ""
        }
    }
}

What we are doing here, is adding an autoload for App namespace to point to our project’s root.

Run:
composer install

You should see a new folder called vendor

If you navigate to: projectA/vendor/composer/autoload_namespaces.php, you will see the namespace ‘App’ that we’ve just created.

Now, we can create our project.

I will be using public/index.php to bootstrap my project.

So now inside the root folder, we can create Bootstrap.php

Because we have already set our namespace in the autoloader to be App, we can use it here.

Apart from autoloading, composer’s true power comes from easily installing external packages.

To look into that, add the package you would like to install by editing composer.json, and add it within require.

For this example I will use a nice package that uses the two-step-view pattern.

After updating composer.json it looks like this:


{
    "require": {
      "aura/view": "dev-develop"
    },
    "autoload": {
        "psr-0": {
            "App": ""
        }
    }
}

To install, run the following from your root directory:

composer update

With the above command, the aura-view package has now been downloaded within the vendors directory and the autoloader has been updated accordingly.
